# M0.3 — Region inference, first sound increment (traversal cursor)

*Not the full monthly build-out — the single sound, measured step that T14
sharpened: eliding the HOT traversal call sites at compile time. It closes ~half
the gap to the oracle ceiling and keeps Java at 65/65.*

## The extension (backend `borrow_slots`)
A `GetField` load (`cur = cur.next`) used to produce an owning ref-local →
retain/release per loop iteration (the hot sites from M0.1c/T14). New: such a
local is a **borrow** (no RC) when — soundly — all of the following hold:
1. the loaded field is written NOWHERE in this function (`PutField`), AND
2. the base is stable (borrowed parameter or itself a borrow slot), AND
3. the function calls NO user function (only `jrt_*` intrinsics, which touch no
   user fields) → the function-local PutField check is thereby
   complete (no callee can change the field beneath us).

The field keeps the value alive, the base is anchored to the stable root
parameter (which the caller holds), the field is immutable during the function
→ the cursor may borrow instead of count. Escape (store/return of the cursor)
retains as before at the store site itself → sound even then.

## Effect (pagerank, best-of-7, -O2)
retain/release sites in the pagerank LLVM: **59 → 37** (the loop cursors gone).

| N | col-ON before | col-ON after | RC-only before | RC-only after | Oracle |
|---|---|---|---|---|---|
| 64 000 | 4.5× | **1.6×** | 2.0× | **1.3×** | 1.1× |
| 256 000 | 4.3× | **2.0×** | 2.2× | **1.5×** | 1.1× |

**~Half the gap to parity closed, with ONE sound pattern.** Bit-identical
correct (82040). Confirms T14 empirically: the hot call sites carry the cost,
and compile-time-eliding them works.

## Honestly: what this is and what it is NOT
- **Is:** a sound, measured first step; Java 65/65 unchanged (the conservative
  "no user calls" guard disables it there mostly where callees could change
  fields). It is exactly the `this`-borrow idea of the backend, extended to
  traversal cursors over immutable fields.
- **Is NOT:** general region inference. The "no user calls" guard is coarse —
  a function that calls other user functions (the normal case in real code)
  does NOT get the elision. The next step would be **interprocedural field-write
  analysis** (which fields does a callee write transitively?), analogous to the
  existing `static_writes` fixpoint for static fields — then it also applies to
  functions with calls. That is the monthly part.
- **Remaining gap 1.3–1.5× → 1.1×:** remaining RC sites (field bases that do not
  satisfy the guard) + codegen. Loop-stable remains the target case;
  topology-mutating remains RC.

*Repro: `vire build [--no-cycles] -o BIN benchmarks/vire-m0/pagerank.vr`.*

## v2 — interprocedural (addendum)
The v1 guard "no user calls" is replaced by an **interprocedural field-write
analysis** (`instance_field_writes`, call-graph fixpoint): a `GetField` borrows
when the function AND its transitive callees do not write the field and no opaque
call (virtual/poly/extern) can change it. This way the elision also applies in
functions that call other (non-writing) user functions.
**Effect pagerank 256k: col-ON 2.0×→1.55×, RC-only 1.5×→1.25× Rust** (ceiling 1.1×).
binary-trees unchanged (allocation-, not traversal-bound). Java 65/65.

## v3 — the ceiling is reached (addendum, measured)
Fresh measurements (best-of-5, `-O2 -march=native`) show: **region inference has
fully closed the RC gap on the traversal path.** The v2 value "1.25× RC-only" was
an earlier state; currently `normal == --no-rc` (the ceiling).

pagerank 262144, Vire normal vs. Vire `--no-rc` (RC fully off) vs. Rust indices:
- **iters=40:**  normal 0.042 / **--no-rc 0.042** / rust 0.022 — normal == --no-rc
  (RC traffic = 0 noticeable); the 1.9× to Rust are **pure allocation** (262144
  `jrt_alloc`/malloc vs. Rust's three bulk `vec![…; n]`), since turning RC off
  changes NOTHING.
- **iters=400:** normal 0.231 / --no-rc 0.224 / **rust 0.223** — the clean
  traversal loop dominates → **parity (1.03×)**; normal vs --no-rc = 3% (that is
  build_rings' one-time owned RC, now a small fraction).

Evidence in the emitted IR (`--emit-llvm`): the hot pagerank inner loop AND the
recursive read-only tree `sum()` carry **zero retain/release** — `cur.next`,
`cur.prev`, `cur = cur.next`, `t.left`/`t.right` are all borrows.

**Honest overall verdict:** on traversal-heavy code there is **no more RC to
elide** (`normal == --no-rc`). The remaining distance to Rust/C++ is the
**allocator** (malloc-per-node vs. bulk/arena) — orthogonal to region inference,
addressable via capsule/arena, not via further borrow analysis. Additional
cleanup: `retain(null)` elision in the backend (provable no-op; cleaner IR, no
measurable speedup, since RC is already at the ceiling). binary-trees: normal
0.477 vs --no-rc 0.468 = ~2% RC — the 2.7× gap to C++/Rust is likewise the
allocator.
